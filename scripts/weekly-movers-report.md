## Technical Notes

### score_history table status
The `score_history` table exists in `scripts/d1-schema.sql` but has **not been applied to the remote D1 database** (`agentrank-db`, ID: `946abec6-5f2b-4177-b1cb-b84769a20883`). The cron-snapshot worker writes to this table for sparkline data. A migration is needed:

```sql
CREATE TABLE IF NOT EXISTS score_history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  tool_slug TEXT NOT NULL,
  tool_type TEXT NOT NULL DEFAULT 'tool',
  score REAL NOT NULL,
  stars INTEGER NOT NULL DEFAULT 0,
  freshness_score REAL,
  issue_health REAL,
  contributors INTEGER NOT NULL DEFAULT 0,
  dependent_score REAL,
  recorded_at TEXT NOT NULL DEFAULT (date('now')),
  UNIQUE(tool_slug, tool_type, recorded_at)
);
CREATE INDEX IF NOT EXISTS idx_sh_slug ON score_history(tool_slug, tool_type, recorded_at);
CREATE INDEX IF NOT EXISTS idx_sh_date ON score_history(recorded_at);
```

Run via: `npx wrangler d1 execute agentrank-db --remote --command "<above SQL>"`

### rank_history vs movers.json discrepancy
The rank_history table in remote D1 shows identical scores across both snapshots (no deltas). This is because the local `data/agentrank.db` scored tools differently — the full ranked.json (25K tools) was scored with a different weight profile than the initial D1 seed. The movers data in this report reflects the local SQLite comparison which is the more meaningful dataset.
