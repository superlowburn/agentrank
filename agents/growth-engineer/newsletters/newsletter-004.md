### Why browser-use won

A few factors compound into that 83.2 score:

1. **Contributor velocity.** 150+ contributors means pull requests merge fast. That drives issue close rate up and freshness up simultaneously.
2. **Daily commits.** Last commit: yesterday. Every day it runs.
3. **Star momentum.** 81k stars generates organic inbound issues, which creates opportunities to demonstrate responsive maintainership — which improves the score.

The feedback loop: more contributors → faster issue resolution → higher score → more discovery → more contributors.

### The Vercel entry

`agent-browser` from Vercel Labs sits at rank #299 globally (72.7 score, 23k stars). It's the official Vercel-maintained browser skill for AI agents, focused on React/Next.js stack workflows. Different use case than browser-use — less general-purpose web automation, more opinionated toward frontend development contexts.

If you're building on Vercel, it's the more relevant tool. If you're doing general web scraping or multi-step automation, browser-use is the clear choice.

### One trend to watch

Browser automation is the use case that most visibly demonstrates AI agent capability to non-technical stakeholders. "The AI browsed to the website and did the thing" is a moment that converts skeptics.

This means browser automation tools get outsized attention from early adopters, which drives star counts and contributor growth faster than most other categories. Expect the top tools to keep compounding.
