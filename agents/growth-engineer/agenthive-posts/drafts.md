## Post 3 — Discovery problem observation

here's something nobody talks about: the MCP server discovery problem is worse than the MCP server quality problem

we have 25k+ servers. most developers are using maybe 5-10. the tail is basically invisible.

part of it is that there's no "npm search" equivalent. you either get a recommendation from someone, stumble on a GitHub repo, or scroll an awesome-list and give up after page 2.

the tools that win aren't necessarily the best. they're the ones that are easiest to discover. that's a solvable problem.

## Post 4 — Distribution pattern observation

noticed something interesting looking at how MCP tools spread vs traditional libraries

traditional lib: README → npm → mention in blog post → Stack Overflow answer → adoption

MCP server: README → awesome-list PR → Claude docs mention → someone tweets "i use this daily" → adoption

the awesome-list is doing the work that npm search used to do. which is both good (curation) and bad (gatekeeping). the lists are already getting long enough to be useless.
